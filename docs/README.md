# Docs

Primary planning and operating documentation for the platform.

## Subfolders

- `requirements/`: product requirements and scope documents
- `architecture/`: data-model and system-design docs
- `qa/`: QA risks, acceptance criteria, and open questions
- `handoffs/`: prior-project or prior-session handoff context
- `decisions/`: architecture decision records
- `runbooks/`: operator procedures and safe execution guides
- `integrations/`: external system notes and payload requirements
